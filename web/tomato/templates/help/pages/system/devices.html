Devices

Introduction

A device is a virtual machine based on either KVM or OpenVZ or RePy.

Every device is based on a device template, and associated to a device profile for resource management.

Since a topology is a network of connected

devices, devices are the core functionality of ToMaTo.

Preparation

Before a device can be run, it must first be prepared. This involves allocation of diskspace, deployment of the template and all the configuration which has to be done on its host.

ToMaTo uses load balancing to deploy devices on several hosts.

Devices can be destroyed to undo the preparation.

Run

A running device uses resources as specified by its device profile.
